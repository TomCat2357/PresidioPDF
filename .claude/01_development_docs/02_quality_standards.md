# コード品質基準

## Zero Warning Policy
プロジェクトではゼロワーニングポリシーを採用。すべての警告は修正されるべきエラーとして扱う。

## 型安全性
- 厳密なTypeScript型付けのPython版として、mypy型チェックを厳格に適用
- `Any`型の使用は禁止（特別な理由がない限り）
- すべての関数は型ヒントを必須とする

## コード品質チェック
品質チェックは以下のコマンドで実行：
```bash
uv run mypy src/          # 型チェック
uv run black --check .    # フォーマットチェック
uv run pytest --cov=src  # テスト・カバレッジチェック
```

## テスト駆動開発 (TDD)
- 新機能実装前にテストケースを先に作成
- 単体テスト・統合テスト・E2Eテストを段階的に実施
- カバレッジ80%以上を維持

## 開発ワークフロー品質チェック
```bash
# 総合品質チェックコマンド
uv run black . && uv run mypy src/ && uv run pytest --cov=src
```

すべてのPRは上記チェックをパスすることが必須条件。
